# Brute Force Detection Using Splunk

## Overview

This project focuses on detecting brute force login attempts using Splunk. A sample authentication log dataset was analyzed to identify failed logins, suspicious IP addresses, credential stuffing attempts, compromised accounts, and account lockouts.

## Objectives

* Analyze authentication logs
* Detect brute force attacks
* Identify suspicious IP addresses
* Investigate potential account compromises
* Create alerts for future detection

## Skills Used

* Splunk
* SPL Queries
* Log Analysis
* Security Monitoring
* Incident Investigation
* Dashboard and Alert Creation

## Key Findings

* Multiple failed login attempts were identified from suspicious IP addresses.
* Credential stuffing activity was detected using username analysis.
* Potential compromised accounts were discovered through failed-to-successful login patterns.
* Off-hours login activity and account lockouts were investigated.

## Project Outcome

This project helped strengthen practical SIEM skills, SPL query writing, log investigation techniques, and SOC analyst workflows used in real-world security operations.
